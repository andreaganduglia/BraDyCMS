    How to create a new menu?
    Main menu > Menu > Add new menu item
    Fill the field "Menu" with the name of the new menu.
    What information are needed to create a new menu item?
    To add a new menu item the following information is necessary:
    • Text: is the text that users will see
    • Link: the link the menu item will point to. For external links also the http:// part is needed (eg.: http://bradypus.net). For internal links the following syntaxt should be used:
      • ./art_title=article_text_id for links to articles
      • ./section=section_name for links to section blogs
      • ./ to link to the home page
      • # do nothing link
      Links to articles, section blogs and more can also be entred by using the widget on the left. Clicking on one article or section, etc, the input will be filled automatically.
    • Target: where the link will be opened:
      • if not set all links will be opened in the same window or tab
      • _blank will open the link in a new window or tab
    • Title: an explicative text that the user will see on hovering the menu text with the cursor
    • Menu: menu name. If choosen from the available list, the menu item will be added to that menu. If a new string is entered a new menu is created. Attention: No white spaces, upper case or special characters in this field, please!
    • Sub menu of: if an existing menu is select a list of the available menu items (siblings) will be shown. If one of them is selected this menu item will be nested to the selected item. Attention: Please see the "How to manage menus?" section for easily nestins menu items
    • Sorting: Only integers are allowed in this field. Use this field to controle the order of appearance of the menu items. Attention: Please see the "How to manage menus?" section for easily sorting menu items
    How to manage menus?
    Main menu > Menu > Show all menus
    If menus are available will be listed here. Fot each available menu will be provided a list containing all items. Clicking on an item will open it in a new tab for editing (for detail see What information are needed to create a new menu item?).

    Menu items can be ordered bu dragging & dropping them. In the same way these items can be nested into one another. Settings are automatically saved.

    Can I have nested articles?
    BraDypUS CMS supports something similar to "Nested articles". It's a feature that permits to use an article as an introduction text for a list of articles, usually articles of the same section.     How can I add a galley in an article's body?

    To add a gallery in the article body a special syntax is provided:

    [[gallery]]gallery_name[[/gallery]]

    Where gallery_name is the name (lower-case, no-spaces, no special characters) of the gallery. This will display a div having css class gallery-container and containing an unordered list (ul) with images and captions. For more details on output check htmlOut.gallery() documentation.

      Some formatting options are available: 
    • [[gallery class="my_css_class"]]gallery_name[[/gallery]]
      my_class_name is a css class that will be added to the unordered list containing the gallery. Manu class names can be added, separated by spaces.
    • [[gallery rel="true"]]gallery_name[[/gallery]]
      if rel option is present (and set to true or anything else) the gallery elements will be linked to each other, otherwise each image will be considered as an indipendent gallery.
    A full example is as follows: 
    [[gallery class="class1 class2" rel="true"]]gallery_name[[/gallery]]
    What is a conditional loading gallery and how can I add it in an article's body?

    BraDyCMS permits the creation of conditional loading galleries. Using these kind of galleries instead of the normal ones permits web designres to create faster websites for mobile templates

    If you are building responsive layouts and do not want to show your image gallery in smaller devices, you surel do not want to load gallery image at all, in order to save bandwidth.

    Conditional loading galleries will load images only if the unordered list (ul) element which holds the galley itself is visible. Images will be loaded using javascript. A jQuery plugin (jquery.cl_gallery.js) is provided.

    To add a conditional loading gallery in the article body a special syntax is provided:

    [[cl_gallery]]gallery_name[[/cl_gallery]]

    Conditional loading galleries uses the same options as galleries

    Hold on! For conditional loading image galleries to work properly adeguate javascript jquery plugin jquery.cl_gallery.js should be loaded and executed!

    How to add a Twitter widget in an article's body?

    To add a Twitter widget you must create it before in Twitter website: https://twitter.com/settings/widgets. Afterwords you can add it in your content using this simple tag:

    [[twitter user="twitter_username" id="twitter_widget_id"]]Some intro text[[/twitter]]

    Where
    twitter_username (*required) is your Twitter username;
    id (*required) is your widget id, provided by Twitter and
    some intro text is the text that will be shown id something goes wrong nad Twitter widget is not initialized.

    For DEVELOPERS: how to create custom modules?

    BraDyCMS allows users to create custom modules that can be called and loaded directly form articles content.

    Modules can contain PHP and/or HTML and/or JAVASCRIPT code and are simple to build just as creating PHP class with static functions

    To create a module called my_module just create a PHP file in ./sites/default/modules (if this folder does not exist it can be created by the user). The file must have the same name of the module followed bu the extension .inc, eg. ./sites/default/modules/my_module.inc. This file must contain a PHP class having the same name as the module and the file itself, eg. my_module. The class can hame many methods, all the methods you need, but one method is mandatory, the public, static method init. The init method will be run automatically when the module is called.

    Calling a user module from inside an articles is as simple as including in the content a simple tag with the name of the module, eg.:

    [[my_module param1="param1_value" param2="param2_value"]][[/my_module]]

    You can add all the parameters you need to your module, adding attributes to your tags. These parameters will be available in the init method of your module class as an associative array. You can include any PHP code in your init method, call other methods of the same class or use system functions. You can include HTML, CSS and JAVASCRIPT.

    Warning: The init method shoud NEVER echo or print content. The HTML, CSS, JAVASCRIPT to include in the page should be returned!

    You're done!
